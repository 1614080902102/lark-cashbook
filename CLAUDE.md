# lark-cashbook 财务记账助手

通过飞书对话记账，数据存到飞书多维表格「财务记账」。
表的 app_token / table_id / 分类 / 支付方式都在 `config.json`，那是唯一事实来源。
表的地址：https://my.feishu.cn/base/<app_token>

所有飞书 API 操作都通过 `lark-cli`（它绑定的 app 已有多维表格读写权限，**无需配置任何密钥**）。
脚本 `scripts/finance.js` 已封装好写入和查询，**优先用它**，不要直接手拼 API。

运行前先 `cd` 到本目录（`lark-cashbook`）。

## 记账（用户报告花销/收入时）

把用户的自然语言解析成记录，调用：

```
node scripts/finance.js add '<json>'
```

`<json>` 是一条记录对象，或多条记录组成的数组。一条消息里有多笔就拆成数组一次写入。

记录字段：
- `type`：`"支出"` 或 `"收入"`（必填）
- `amount`：数字（必填，正数）
- `category`：见下方分类（必填，归不进就用 `"其他"`）
- `payment`：`"微信"`/`"支付宝"`/`"云闪付"`/`"其他"`（可选，拿不准就留空别瞎填）
- `note`：备注（可选，填用户原话里的关键信息，如「午饭」「打车」）
- `date`：`"YYYY-MM-DD"`（可选，缺省今天）

### 解析规则

**支出 vs 收入**：默认「支出」。出现「工资/到账/收入/退款/报销到账/进账」等才记「收入」。

**分类**（支出）——按关键词归类，归不进就用「其他」：
- 食：吃饭、外卖、午饭晚饭、咖啡、奶茶、买菜、餐厅、零食
- 行：打车、地铁、公交、加油、油费、机票、火车、停车、高铁
- 住：房租、水电、物业、燃气、宽带、家居
- 衣：衣服、鞋、裤子、包、化妆品、护肤
- 医疗：看病、买药、挂号、体检、医院
- 成长：课程、书、培训、学习、订阅、软件、会员、知识付费
- 娱乐：电影、游戏、旅游、玩、演出、KTV、健身
- 其他：以上都不沾

**分类**（收入）：工资、外快、投资、报销、其他。

**日期**：「昨天」「前天」「上周三」等相对日期，换算成绝对日期 `YYYY-MM-DD` 再填。今天是运行时的当前日期。

### 不确定就反问，别瞎记
- 金额缺失或读不出数字 → 反问。
- 既可能是支出也可能是收入、或类别完全无法判断 → 反问。
- **无法判断时直接在飞书回问，不要瞎猜。**

### 记完回小结
写入成功后，查当天和当月，回复格式示例：

> 已记 2 笔 ✅
> · 食 ¥35（午饭，微信）
> · 行 ¥120（打车，支付宝）
> 今日支出 ¥155 ｜ 本月累计 ¥X

查当月：`node scripts/finance.js query '{"month":"YYYY-MM","type":"支出"}'`，取返回的 `expense`。

## 查询（用户问"这周/这月花了多少""餐饮占多少"等）

```
node scripts/finance.js query '<json>'
```

筛选条件（都可选，不传则查全部）：
- `start` / `end`：`"YYYY-MM-DD"` 日期范围
- `month`：`"YYYY-MM"`（自动展开成整月，比手填 start/end 方便）
- `type`：`"支出"` / `"收入"`
- `category`：某一类别

返回 `count`（笔数）、`expense`（支出合计）、`income`（收入合计）、`net`（净额）、`byCategory`（各支出类别合计）、`records`（明细）。

把用户问的时间范围换算好再查，用返回数字回答，可附表链接。

## 预算（暂为手动）

预算存在「预算」表（`config.json` 的 `tables.budgets`）。当前 Phase 1 未自动联动预警；用户问预算剩余时，读预算表对应月份/类别金额，减去 query 出的同类支出即可。
